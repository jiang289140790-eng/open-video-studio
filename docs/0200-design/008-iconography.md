## Purpose

Define the iconography language for Open Video Studio.

## Requirements

- Use icons to improve recognition, scanning, and control clarity.
- Provide accessible labels for icon-only controls.
- Keep icons consistent and non-decorative.

## Scope

Icons should make controls faster to recognize and interfaces easier to scan. They should not become decorative illustration.

Icon roles:

- Navigation.
- Toolbar actions.
- Object types.
- Status states.
- AI activity.
- Review and approval states.
- File, media, and timeline actions.
- Integrations and external destinations.

Usage rules:

- Prefer familiar symbols for common actions.
- Pair icons with labels when meaning is not immediately universal.
- Use tooltips or accessible names for icon-only controls.
- Keep icon weight, optical size, and alignment consistent.
- Do not use icons as a substitute for clear information architecture.
- Status icons must work with text and color semantics.

## Acceptance Criteria

- Icon usage improves recognition or scanning.
- Icon-only controls are accessible.
- Icons remain visually consistent across product areas.
- AI-related icons do not overpromise autonomy or magic.

## Future Evolution

Future implementation may select an icon library, but this document defines usage standards independent of the library.

## Future Plan

Define approved icon set, custom product icons, and object-type icon taxonomy when implementation planning begins.

## AI Context

When suggesting controls, choose conventional icon meanings before inventing new symbols. Add labels when ambiguity would slow users down.
