## Purpose

Own strategic sequencing, phases, milestones, launch gates, release planning, and long-term platform direction.

## Requirements

- Roadmap items must reference product, architecture, or operational source documents.
- Milestones must define scope, success criteria, risks, and dependencies.
- Roadmap changes that alter technical direction may require an ADR.
- Production milestones must define launch gates for security, testing, observability, deployment, rollback, disaster recovery, and cost control.
- Roadmap sequencing must identify foundational dependencies before user-facing commitments.

## Acceptance Criteria

- Teams can understand what to build next and why.
- Sequencing is tied to documented requirements and constraints.
- Launch readiness can be evaluated before public commitments are made.
