## Purpose

Store shared project assets such as brand references, screenshots, diagrams, research exports, and generated media used by documentation or planning.

## Requirements

- Assets must be named clearly and grouped by purpose when folders are added.
- Source or license context should be recorded where relevant.
- Do not store secrets or private user data here.

## Acceptance Criteria

- Documentation can reference stable assets.
- Contributors can understand what an asset is and why it exists.
